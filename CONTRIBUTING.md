# Contributing

Contributions are welcome when they support defensive learning and secure coding.

## Contribution Rules

- Keep all content educational, defensive, and authorized-use only.
- Do not add exploit-ready payload collections.
- Do not add credential theft, token theft, cookie theft, or data exfiltration examples.
- Do not add bypass chains, evasion chains, or offensive infrastructure.
- Add clear documentation explaining how an example prevents or reduces XSS risk.
- Prefer safe examples that demonstrate output encoding, safe DOM APIs, templating safety, validation, sanitization, and Content Security Policy.
- Use controlled local examples only.

## Review Standard

Before submitting a change, verify:

- The content is defensive.
- The example is safe to publish.
- The wording does not encourage unauthorized testing.
- The example explains prevention or secure design.
